# Alphapai Reptile Skill (金融投研抓取引擎)

本系统是一个基于 Python 与 MySQL 构建的企业级自动化金融数据抓取与持久化平台。结合 OpenClaw AI 核心（小龙虾），能够实现全自然语言交互式的环境配置、关键词管理与任务调度。

本系统专为解决高难度金融平台数据抓取设计，具备应对"接口伪分页"、"随机ID漂移"、"长效Token续期"等复杂场景的能力。

---

## ✨ 核心功能特点

### 1. 🔄 智能全量与增量平滑切换 (核心特性)

系统底层的 `category_stats` 表自动维护抓取生命周期，彻底释放人工监控压力：

- **全量溯源模式 (Status=0)**：首次添加关键词时，系统会自动比对接口返回的总条数（`total_count_api`）。爬虫会持续翻页，直到将历史积累的所有研报、图表、公告等数据悉数抓取完毕。
- **毫秒级增量断点 (Status=1)**：当全量数据抓取达标后，系统自动将该分类切换为"增量监控状态"。在增量模式下，爬虫每次只从第 1 页开始扫描，一旦遇到数据库中已存在的 `post_id` 或 `title`，立即触发"增量阻断"，停止当前分类的无效翻页，极大地节省了系统资源并降低了风控风险。

### 2. ⚡️ 列表与详情解耦，支持完全并行处理 (核心特性)

本系统的架构设计采用了经典的"生产者-消费者"模型，列表爬取脚本与详情爬取脚本互不干扰，完全可以同时并发运行：

- **生产者 (`list_crawler.py`)**：只负责高频遍历接口列表，获取标题、发布时间和 ID，并以极快的速度写入 `data_list` 表（初始状态 `detail_status = 0`）。
- **消费者 (`detail_crawler.py`)**：负责重负载的工作，专门从数据库中查询 `detail_status = 0` 的记录进行深度解析和附件下载。完成一条标记一条（`1` 为成功，`2` 为失败）。

**并行优势**：由于两者操作的数据边界清晰（列表管插入，详情管更新），你可以将它们挂在两个不同的后台进程（或不同的 Cron 定时任务）中同时跑，列表脚本在前面探路，详情脚本在后面消化，成倍提升数据入库吞吐量。

### 3. 🧠 动态多模态解析与 AI 纪要整合

- 自动识别并提取 `aiSummaryV3` 结构化 AI 摘要。
- 自动解析嵌套极深的 `mtSummarySwitchOpen` 人工会议纪要。
- 无缝处理图表接口（以 `reportId` 作为图表唯一标识映射）。
- 自动剥离 HTML 标签，提取干净的正文并分离内嵌图片。

### 4. 📂 高清晰度文件归档

彻底抛弃无意义的 Hash 命名。所有下载的图片、PDF 严格遵守本地化路径策略：

```
DATA_ROOT / 监控关键词 / 所属分类 / 归档日期 / 文章标题_发布日期.后缀
```

> 例如：`files/德福科技/研报/2026-05-11/电子行业跟踪报告_2025-12-24.pdf`

## ⚙️ 环境配置与依赖安装

请确保您的系统环境已安装 **Python 3.8+** 及 **MySQL 5.7+**。

### 1. 安装依赖

```bash
pip install requests pymysql beautifulsoup4 lxml python-dotenv
```

在项目根目录创建 `.env` 文件，并填入以下参数：

```ini

DB_USER=root
DB_PASS=your_password
DB_NAME=alphapai_db
DB_PORT=3306

# 登录账户配置
LOGIN_MOBILE="+86156xxxxxxxx"

# 任务控制与防风控休眠 (单位: 秒/分钟)

RUN_DURATION_MAX=20
LINK_SLEEP_TIME_MIN=2
LINK_SLEEP_TIME_MAX=5
QUIET_START=00:01

```

## 🤖 OpenClaw "小龙虾" AI 助手使用指南

本系统完美适配 OpenClaw 技能挂载，您无需编写任何代码或操作数据库软件，直接通过自然语言即可完成所有调度。

### 🔑 第一步：让小龙虾帮你完成安全认证

首次使用时，需将你的加密密码载荷写入数据库：

1. 浏览器打开目标平台按 `F12`，在网络(Network)中找到 `accountLogin` 接口。
2. 复制 Payload 中的 `password` 加密长串。
3. 对小龙虾说：

> "小龙虾，这是我的登录密码加密载荷 [粘贴密文]，请帮我写入数据库的 `auth_credentials` 表中，账号是 [你的手机号]。"

### 🗣️ 第二步：自然语言管理关键词

直接对小龙虾下达监控指令：

> "小龙虾，请帮我在抓取监控表 `crawl_keywords` 里加上【光模块】和【宁德时代】，记得把状态设为激活。"

### 🚀 第三步：触发任务执行

通过小龙虾或终端命令行执行：

#### 环境初始化 (首次必执行)

> "小龙虾，帮我初始化一下爬虫环境建表。"

或执行：

```bash
python3 init_setup.py
```

#### 触发列表扫描 (生产者)

> "小龙虾，帮我去扫一遍列表，更新一下总数。"

或执行：

```bash
python3 list_crawler.py
```

#### 触发深度解析与附件下载 (消费者)

> "小龙虾，把列表里还没解析的正文和 PDF 都下载下来。"

或执行：

```bash
python3 detail_crawler.py
```

为了保障数据安全，赋予小龙虾系统级约束：

- **防误删护栏**：系统严禁通过对话指令删除、移动或篡改存放在 `DATA_ROOT` 目录下的任何 `.pdf`、`.png` 等附件资源。

## ❓ 常见问题 FAQ

### Q1：为什么列表抓取时，有时日志显示"发现重复数据，触发增量阻断"？

**A：** 这是本系统增量模式生效的标志。当脚本在第一页发现了数据库中已经存在的数据（通过 `post_id` 或 清洗后的标题 识别），它会立刻明白"旧数据已经衔接上，无需再往下翻页了"，从而瞬间停止抓取，极大地提高了效率。

### Q2：接口返回了 62 条数据，为什么日志显示只入库了 60 条？

**A：** 这证明了系统双重查重引擎正在发挥作用。部分接口会存在数据冗余（例如同一篇文章被系统推了两次，或者你的不同关键词命中了同一篇研报），底层 SQL 检测到了冲突并自动 `continue` 拦截，这是保障数据干净无脏数据的正常行为。

### Q3：我可以同时运行 `list_crawler.py` 和 `detail_crawler.py` 吗？

**A：** 完全可以，且非常推荐。它们在数据库中操作不同的字段状态，并行运行不会产生死锁或冲突，是提升大批量数据抓取效率的最佳实践。
