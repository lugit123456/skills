# 港交所披露权益（DI）数据同步引擎

## 概述

本 Skill 为港交所披露权益（Disclosure of Interests, DI）数据同步引擎，旨在建立一个具备高可靠性的本地化金融数据库，完整捕获并持久化港交所披露权益平台上的权益变动记录。

## 核心特性

### 1. 自适应双模式同步

引擎支持两种运行模式，根据数据状态自动切换：

- **BULK 模式（历史全量）**：
  - 利用滑动窗口（180 天）追溯自 2003 年起的历史数据
  - 适用于首次部署或需要完整历史数据重建的场景
  - 通过分页与日期窗口组合，确保全量数据无遗漏

- **INCREMENTAL 模式（每日增量）**：
  - 历史补课完成后自动切换
  - 仅爬取最近 3 天的数据
  - 采用"探测到重复项即停"的短路机制，避免无效请求，提升同步效率

### 2. 全字段指纹判定

针对港交所页面中常见的以下特殊情况，引擎采用基于 11 个关键字段的复合指纹策略：

- **无序号行**：部分记录缺少官方序号，无法依赖单一标识去重
- **外观一致修订行**：内容高度相似但存在细微差异（如关联详情 URL 不同）

**指纹生成逻辑**：
提取 11 个关键字段进行组合哈希，只要任一字段存在差异（即使仅为 URL 参数变化），即视为独立记录入库。该机制确保数据完整性，杜绝因表面相似而导致的数据丢失。

### 3. 自动化日志管理

- **滚动文件日志**：日志按运行日期即时切分，文件名包含日期信息
- **自动保留策略**：仅保留最近 7 个日期的日志文件
- **调试友好**：确保调试信息完整可追溯，同时防止磁盘空间无限增长

## 数据流

```
港交所 DI 网页
    │
    ▼
[爬虫层] ──► 原始 HTML 解析
    │
    ▼
[指纹生成层] ──► 11 字段复合指纹计算
    │
    ▼
[去重判定层] ──► 指纹比对 / 重复探测
    │
    ▼
[持久化层] ──► 本地金融数据库写入
    │
    ▼
[日志层] ──► 滚动日志记录（保留 7 天）
```

## 运行模式切换逻辑

```
首次运行 / 强制全量
        │
        ▼
   ┌─────────┐
   │ BULK    │ ◄── 滑动窗口 180 天
   │ 模式    │     追溯至 2003
   └────┬────┘
        │ 历史数据全量完成
        ▼
   ┌─────────┐
   │INCREMENT│ ◄── 每日增量 3 天
   │ 模式    │     遇重即停
   └─────────┘
```

## 技术要点

| 模块 | 说明 |
|------|------|
| 滑动窗口 | 180 天为单位的日期分段，平衡单次请求负载与数据完整性 |
| 复合指纹 | 11 个关键字段组合哈希，精确区分相似记录 |
| 短路机制 | 增量模式下探测到连续重复即终止爬取，节省资源 |
| 日志滚动 | 按日期切分 + 7 天保留策略，兼顾可追溯性与磁盘管理 |

## 适用场景

- 需要本地化维护港交所披露权益完整历史数据的量化研究或合规系统
- 对数据完整性要求极高，不能容忍因页面展示问题导致的数据遗漏
- 需要自动化日常同步且对运维成本敏感的生产环境
